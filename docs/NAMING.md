# Naming — Top 10 Candidates

## What this product is

A **universal content-to-markdown converter and personal knowledge base** built for AI agents. Save any URL, file, YouTube video, or note — get clean, structured markdown back. Chrome extension for capture, CLI + MCP for agent access. Fabric-style LLM extraction turns raw content into structured knowledge (ideas, insights, quotes, claims, references).

**One-liner:** "Save anything. Get markdown. Agents read it."

---

## Top 10

### 1. **markstash**
> *Stash it. Mark it down.*

- **Description:** Save anything as markdown — the knowledge stash for AI agents.
- **GTM:** Developer tool positioning. "Your AI agent can't read your bookmarks. markstash fixes that." Launch on HN as open-source CLI tool. Chrome extension drives organic adoption — every save is a markdown file agents can search.
- **Strengths:** Clear (mark + stash), available on npm, short CLI (`markstash add`), already built and shipped.
- **Weakness:** Slightly generic. Another dev already has a 1-star repo with the name.

### 2. **mdstash**
> *Your markdown stash.*

- **Description:** Convert and stash anything as markdown. CLI, MCP, Chrome extension.
- **GTM:** Same as markstash but leans harder into the `.md` file format angle. "Everything becomes an .md file." Appeals to Obsidian/note-taking crowd.
- **Strengths:** Very short CLI typing (`mdstash add`), no existing GitHub repos, available on npm. Clear what it does.
- **Weakness:** Less memorable than a word — it's an abbreviation.

### 3. **inkwell**
> *Fill your well. Draw from it.*

- **Description:** A knowledge well that converts any content into agent-readable markdown. Save URLs, videos, files — draw insights from them later.
- **GTM:** Premium positioning. Inkwell evokes writing, knowledge, permanence. Market as "the knowledge base for the AI era" — not another bookmark manager, but a well of extracted wisdom your agents draw from.
- **Strengths:** Beautiful name, memorable, evokes ink/writing/knowledge. Available on npm? (Need to verify — `inkwell` might be taken, but `inkstash` is available.)
- **Weakness:** Doesn't immediately suggest "markdown" or "save."

### 4. **glean**
> *Glean the good parts.*

- **Description:** Glean extracts the valuable knowledge from any content — URLs, videos, files — and stores it as searchable markdown.
- **GTM:** The extraction angle. "Don't just save links — glean the wisdom from them." Positions the LLM extraction (Fabric-style patterns) as the hero feature, not just the storage.
- **Strengths:** Perfect English word for what the tool does (to glean = to gather useful information). Short, elegant.
- **Weakness:** Already a well-known enterprise product (Glean.com — enterprise search). GitHub org taken. Would live in its shadow.

### 5. **markhoard**
> *Hoard everything. In markdown.*

- **Description:** The markdown hoarder — compulsively saves and converts everything into agent-ready .md files.
- **GTM:** Lean into the "save everything" behavior. "You're already hoarding tabs. Markhoard converts them into something useful." Playful, slightly self-aware positioning.
- **Strengths:** No GitHub org, no npm package, no existing projects. Unique. Playful tone. Clear compound word.
- **Weakness:** "Hoard" has slightly negative connotations (hoarding disorder). Longer to type.

### 6. **savemark**
> *Save it. Mark it down.*

- **Description:** One-click save to markdown. URLs, videos, PDFs, notes — all become searchable .md files for you and your AI agents.
- **GTM:** Consumer-friendly angle. "Pocket saves links you never read. Savemark converts them into knowledge." Chrome extension is the hero — click to save, agents can find it.
- **Strengths:** Intuitive verb+noun. Immediately clear what it does. Good for non-developer audience too.
- **Weakness:** Slightly bland. Could sound like a generic bookmark tool.

### 7. **markfolio**
> *Your markdown portfolio.*

- **Description:** A curated portfolio of everything you've saved, converted to markdown. Search, extract, and feed to AI agents.
- **GTM:** Position as a "second brain" for the AI age. Not just bookmarks — a folio of extracted knowledge. "Build your markfolio. Your agents will thank you."
- **Strengths:** No GitHub repos, no npm package. Unique. "Folio" evokes curation, quality, a collected body of work.
- **Weakness:** "Portfolio" might suggest design/art. Longer name.

### 8. **pith**
> *Get to the pith.*

- **Description:** Pith extracts the essential substance from any content. URLs, videos, files — stripped down to the core ideas, stored as markdown.
- **GTM:** Extraction-first positioning. "Pith doesn't just save — it extracts." The LLM extraction patterns are front and center. "What's the pith of that 2-hour podcast?"
- **Strengths:** 4 letters. Beautiful English word (pith = the essential part). Unique in the tool space. Fast to type (`pith add`).
- **Weakness:** Obscure word — many won't know it. Might need explanation.

### 9. **agentfeed**
> *Feed your agents.*

- **Description:** The content pipeline for AI agents. Save URLs, extract insights, serve structured markdown via CLI and MCP.
- **GTM:** Pure agent-native positioning. "Your agents are starving for context. Feed them." Markets directly to the AI coding agent crowd — Pi, Claude Code, Cursor users. MCP server is the hero feature.
- **Strengths:** Immediately clear value prop for the target audience. No GitHub org or npm package.
- **Weakness:** Narrow positioning — sounds agent-only, might alienate human users who just want a bookmark tool.

### 10. **mdvault**
> *Your markdown vault.*

- **Description:** A secure local vault of markdown knowledge. Everything you save gets converted, extracted, and stored as .md files on disk.
- **GTM:** Privacy + local-first angle. "No cloud. No database. Just .md files in your vault." Appeals to privacy-conscious developers and the local-first movement.
- **Strengths:** Clear metaphor (vault = safe storage). Short. "md" immediately signals markdown.
- **Weakness:** "Vault" suggests security/encryption more than knowledge. Obsidian already owns the "vault" concept in the markdown space. A few small existing repos.

## My Recommendation

**markstash** or **pith** — depending on personality.

- **markstash** if you want clarity and zero explanation needed. It's already built, shipped, and working.
- **pith** if you want something distinctive that positions the extraction as the core value. Four letters, beautiful word, fast to type.
